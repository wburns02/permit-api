Layer 1: Enhanced Table

Transform the static results table into an interactive data grid.

  • Date column — auto-detect date fields (issue_date, date_created, sale_date, etc.), normalize to "DATE" header, format as relative ("2d ago") with full date on hover
  • Checkbox column — first column, select individual rows or "select all"
  • Clickable rows — hover highlight, pointer cursor, click opens slide-out panel
  • Phone icon — if phone number exists in the row data, show a phone icon inline for instant tel: click
  • Row status indicators — subtle color/icon showing if row has phone, has email, has owner name (data quality at a glance)

Layer 2: Slide-out Detail Panel

Right-side panel, ~420px wide, action-first layout.

  • Top 30%: Action Zone — big name/address, massive "Call Now" button (tel: link), phone number displayed large, secondary buttons: "Send to Dialer", "Copy Phone", "View Full Report"
  • Middle: Permit Summary — permit number, type, status, date, valuation, description, contractor info
  • Bottom: Intelligence — lazy-loaded from /analyst/report endpoint: linked permits, violations, sales history, flood zone, demographics. Loads on demand so the panel opens fast.
  • Navigation — up/down arrows or keyboard (j/k) to move between rows without closing the panel

Layer 3: Batch Actions Bar

Sticky bar appears at bottom when 1+ rows are selected.

  • "X selected" count with "Select All" / "Clear" buttons
  • Export CSV — downloads selected rows as CSV
  • Copy to Clipboard — tab-delimited, paste into Sheets/Excel/CRM
  • Send to Webhook — pushes selected rows as JSON to user's configured webhook URL (set in account settings)
  • Send All to Dialer — queues all selected rows with phone numbers to the #dialer page

Layer 4: Webhook Configuration

Minimal settings addition — no complex integration needed.

  • Account settings — "CRM Webhook URL" field where user pastes their Zapier/Make/n8n webhook
  • Test button — sends a sample payload so user can verify it works
  • Payload format — standard JSON array of permit objects, documented in API docs page
  • No auth complexity — webhook URL itself is the auth (standard Zapier pattern)

What's NOT in scope

  • Built-in soft phone (Twilio WebRTC) — that's spec #2
  • Native CRM integrations (GHL, HubSpot, Salesforce) — webhook covers them all for now
  • Call logging/recording — comes with soft phone spec
  • AI call scripts/talking points — future feature
