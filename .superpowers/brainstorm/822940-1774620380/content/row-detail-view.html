When you click a permit row, what opens?

This is the primary interaction — how users drill into a lead

A

Slide-out Panel (Right Side)

Panel slides in from the right covering ~40% of the screen. Table stays visible behind it. User can scan the list and click through rows quickly without losing context.

PERMIT DETAIL

Pros

  • Fast scanning — click through rows without page changes
  • Familiar pattern (Gmail, Salesforce, HubSpot)

Cons

  • Limited width for complex data
B

Full-Screen Detail Page

Clicking a row navigates to a dedicated permit detail page. Full property report with all linked data — permits, violations, sales, liens, flood zones, demographics.

← Back to results   |   FULL PROPERTY REPORT

Pros

  • Room for rich data (maps, linked permits, full history)
  • Can show everything from the /analyst/report endpoint

Cons

  • Loses list context — have to navigate back and forth
C

Expandable Row (Inline Accordion)

Clicking a row expands it in-place, pushing other rows down. Shows key details + action buttons right in the table flow.

Pros

  • Never leaves the table — minimal context switching
  • Simple to build

Cons

  • Limited space for rich data
  • Can feel cramped with action buttons + details
